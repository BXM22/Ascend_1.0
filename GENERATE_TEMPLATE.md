# Generate Template

Reference for **automatic workout template generation** in Ascend: the generate sheet, generation settings, `WorkoutGenerationSettings`, `WorkoutGenerator`, and how `TemplatesViewModel` wires them. For the Templates tab shell and bento library, see [TEMPLATE_VIEW.md](./TEMPLATE_VIEW.md). For global visuals, see [DESIGN.md](./DESIGN.md).

---

## 1. Role in the app

Users can **produce new `WorkoutTemplate` instances** from rules (training phase, equipment, muscle emphasis, day type) without hand-building exercises. Generated templates are **saved to the template library** via `TemplatesViewModel.saveTemplate` and behave like any other saved template (edit, duplicate, start session).

Generation is **local and rule-based** (`WorkoutGenerator` + ExRx catalog); it is not a network LLM call.

---

## 2. Two UI surfaces

| Surface | View | Purpose |
|---------|------|---------|
| **Generate sheet** | `WorkoutGenerationView` | Pick **workout type** (Custom, Push, Pull, Legs, Upper, Lower, Full Body), adjust **training type / goal** (bound to `generationSettings`), run **Generate Workout** or **Quick Generate** (1 / 3 / 5 variations). On success, templates are saved and the sheet dismisses after a short delay. |
| **Generation settings** | `WorkoutGenerationSettingsView` | Deeper tuning: phase guidelines, equipment toggles, muscle-group counts, cardio / calisthenics / warmup / stretch caps, min/max exercises, rest and RIR ranges. Opened from the Templates title **ellipsis** (“generation settings”) and from **Programs** split setup (`WorkoutSplitsSection`). |

Both bind to the **same** `TemplatesViewModel.generationSettings` instance passed from `TemplatesView` (or injected where `templatesViewModel` is shared).

**Templates entry:** `showGenerateSheet` in `TemplatesView` presents `WorkoutGenerationView(viewModel:onStart:)`. **Settings entry:** `viewModel.showGenerationSettings` presents `WorkoutGenerationSettingsView(settings: $viewModel.generationSettings)`.

## 3. Data: `WorkoutGenerationSettings`

**File:** `Ascend/Models/WorkoutGenerationSettings.swift`

**Enums:** `TrainingType` (strength, endurance), `TrainingGoal` (bulk, cut). Together with endurance, these map to **`TrainingPhase`** (bulking / cutting / endurance) for presets and generator logic.

**Notable fields:**

| Area | Fields |
|------|--------|
| Volume | `exercisesPerMuscleGroup`, `minExercises`, `maxExercises` |
| Content flags | `includeCardio`, `includeCalisthenics`, `includeWarmup`, `includeStretch`, `maxWarmupStretchExercises`, `maxCardioExercises` |
| Equipment | `preferredEquipment` (e.g. Bodyweight, Dumbbells, Barbell, Cable, Machine) |
| Session feel | `trainingType`, `trainingGoal`, `restTimeMin` / `restTimeMax`, `rirMin` / `rirMax` |

**`applyPhasePreset()`** updates rest, RIR, and (when a split type is passed) exercise counts from `TrainingPhase`. The generate sheet calls it when training type or goal changes.

**Persistence:** `WorkoutGenerationSettings` is `Codable`. **`TemplatesViewModel`** loads and saves it under **`UserDefaults`** key `workoutGenerationSettings` (`AppConstants.UserDefaultsKeys.workoutGenerationSettings`) with debounced writes when values change. **Reset all data** in Settings clears this key and restores defaults.

## 4. Engine: `WorkoutGenerator` + `TemplatesViewModel`

**Singleton:** `WorkoutGenerator.shared` (`Ascend/ViewModels/WorkoutGenerator.swift`).

It pulls exercises from **`ExRxDirectoryManager.shared`**, filters by **equipment**, **warmup/stretch** settings, excludes **calisthenics category** when building from the directory slice, and uses **day-type** hints (push/pull/legs/upper/lower/full body) for muscle-group filters and stretch relevance. Exercise counts and naming follow phase and split helpers on `TrainingPhase` and internal distribution logic.

**`TemplatesViewModel` wrappers** (all use `generationSettings`):

| Method | Use |
|--------|-----|
| `generateWorkout(name:)` | Custom-style generation; optional name drives `WorkoutDayTypeExtractor` for day-type hints |
| `generatePushWorkout()` / `generatePullWorkout()` / `generateLegWorkout()` | Split day presets |
| `generateUpperWorkout()` / `generateLowerWorkout()` | Upper/lower split |
| `generateFullBodyWorkout()` | Full body |
| `generateWorkoutVariations(count:)` | Multiple distinct templates in one call |

`WorkoutGenerationView` routes the selected **workout type** to the matching method, then **`saveTemplate(_:)`** and success UI.

## 5. Other call sites (same settings)

Anything that needs a generated template can call `WorkoutGenerator` with `templatesViewModel.generationSettings`:

| Area | Typical use |
|------|-------------|
| `WorkoutProgramViewModel` | `ensureTemplateForDay`, `populateProgramDaysWithAutoGeneratedWorkouts`, `autoGenerateTemplate` |
| `WorkoutSplitsSection` | Program creation flow + generation settings sheet |
| `SuggestedWorkoutCard`, `ProgramDayTracker`, `DayTypeInfoCard`, `ActiveProgramDayWorkoutStart` | Generate or ensure template when starting or previewing program days |
